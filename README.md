# public-operator

Planning and execution repo for taking Operator OS from internal prototype to public release.

## Goal

Turn Operator OS into a product that can be shown publicly, onboard real users, and survive a beta without manual babysitting.

## What lives here

- release plan and milestones
- launch blockers and readiness criteria
- workstream docs for product, infra, auth, onboarding, and GTM
- daily status updates
- sub-agent outputs and synthesis

## Current stance

Operator OS looks compelling now, but it is not yet ready for a real public launch. The repo exists to close that gap fast and visibly.

## Core workstreams

1. Product and onboarding
2. Multi-tenant auth and account system
3. Agent reliability and guardrails
4. Managed services integration
5. Billing, limits, and ops
6. Docs, positioning, and launch assets

## Cadence

- daily progress updates
- weekly milestone review
- rolling backlog reprioritization based on blockers

See `docs/public-release-plan.md` for the working plan.
See `STATUS.md` for the running log.
