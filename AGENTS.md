# AGENTS.md

This repository is for planning and executing the public release of Operator OS.

## Expectations

Sub-agents working here should:
- optimize for clear, actionable output
- prefer concrete plans over abstract commentary
- write docs, checklists, specs, issue drafts, and implementation plans
- break large goals into small shippable milestones
- call out blockers, dependencies, and risk early

## Do

- update docs in this repo
- create structured plans and backlogs
- propose milestones and release criteria
- record assumptions explicitly
- keep status notes concise and useful

## Do not

- edit production infra from this repo
- invent features with no launch relevance
- bury key risks in long prose
- treat demos as launch readiness

## Definition of helpful

A good contribution from a sub-agent should make it easier to answer one of these:
- what blocks public beta?
- what can ship this week?
- what is the release sequence?
- who or what owns the next step?
